1. La asignación en Scheme

Hasta esta ahora, no hay forma de cambiar el valor de una variable: 

(define a 23) ; no hay ningún problema

(define a 24) ; assignment disallowed, cannot re-define...

Es curioso que los primeros pasos de cualquier lenguajes es la asignación pero en programación funcional es lo último.

2. Reasignación

Cuando un símbolo esta definido, se puede cambiar su valor utilizando la función set! 

(define a 43)
(set! a 44)

3. Nuevo modelo de computación

La razón de que no se utilice la asignación es que no se puede incluir en la substitución, pero esto trae ventajas. 

(define (crea-contador x)
	(define (siguiente)
		(set! x (+ x 1))
		;; Incrementa el valor de x
		x 			; Devolvemos x
		)
	siguiente
)
(define d (crea-contador 4))
(d) 					; 5

4. El concepto de ambiente

Un ambiente (environment) es una secuencia de marcos (frames). Cada marco es una tabla (posiblemente vacía) de asociaciones, que vinculan nombres de variables con sus valores correspondientes. El primer marco en la secuencia de ambientes es el marco más reciente, y el resto del ambiente se denomina el ambiente envolvente (enclosing environment).

5. Ejemplo de uso (SICP)

(define (make-account balance)
    (define (withdraw amount)
         (begin (set! balance (- balance amount))
                 balance))

    (define (deposit amount)
         (begin (set! balance (+ balance amount))
                balance))

    (define (dispatch m)
          (cond ((eq? m 'withdraw) withdraw)
                ((eq? m 'deposit) deposit)))
    dispatch)

6. Modelo de evaluación de ambientes

En este modelo de evaluación, existe un ambiente global y en ella se evaluan funciones, añadiendo las siguientes reglas:

  • Cada función tiene su ambiente local, que se crea cuando la función \(\lambda\) es evaluada. El ambiente incluye un link al ambiente donde fue creada.
  • Cuando una función es evaluada, se ligan las variables con los argumentos.

7. Modelo de evaluación de ambientes

  • Al evaluar una subexpresión, se utiliza las variables para mirar en el ambiente el valor de cualquier variable.
  • El procedimiento para encontrar una variable en un ambiente es recorrer un árbol

8. Recapitulación

Cuando poniamos procedimientos internos, por ejemplo al definir la función sqrt hay dos ventajas:

  • Los nombres de los procedimientos internos no son alterados por los procedimientos externos.
  • Los procedimientos internos pueden acceder a las variables con las que se han definido.
  • Esto permite la modularización

9. Desventajas del modelo de evaluación

  • Se introduce un ambiente, y la ejecución del programa puede afectar a cosas fuera de su ambiente
  • El orden de ejecución hace que los resultados cambien
  • Se introduce la idea de compartir y la idea de igualdad

10. Orden de evaluación

(define (factorial n)
	(let ((product 1)
              (counter 1))
	  (define (iter)
		  (if (> counter n)
		      product
		    (begin
		     (set! counter (+ counter 1))
		     (set! product (* counter 
				      product))
		     
                     (iter))))
	  (iter)))
